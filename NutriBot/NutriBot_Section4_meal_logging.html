Section 4: Meal Logging & Nutrient Scoring

Before NutriBot can give personalized recommendations, it needs to understand what you’re eating. But users don’t talk in JSON:
“Had biryani, raita, and a Coke for lunch.”
“Added boiled eggs to breakfast.”
“Dinner was salmon, quinoa, and sautéed spinach.”

For the system, this means turning natural language into a structured nutrient log — something queryable, comparable, and analyzable.

The Challenge

This task requires:

  1. Entity extraction (meals, portions, time of day)
  2. Food normalization (map “Coke” to “Coca-Cola, 12 oz”)
  3. Nutrient mapping (via USDA FoodData Central or proprietary tables)
  4. Scoring logic (is this high in sodium? low in fiber?)

Think of it as NER + enrichment → nutrient vector.

Step-by-Step Pipeline

Stage Tool / Strategy
Text parsing spaCy / OpenAI function call-like parsing
Food normalization USDA FDC API + synonym dictionary
Nutrient retrieval API fetch or local SQLite + pandas
Nutrient scoring Custom rule engine or ML classifier

# Step 2: Query USDA FoodData Central for nutrient values
def query_usda(food_item):
    API_KEY = "YOUR_USDA_API_KEY"
    url = f"https://api.nal.usda.gov/fdc/v1/foods/search?query={food_item}&api_key={API_KEY}"
    res = requests.get(url).json()
    top_result = res["foods"][0]
    return {
        "name": top_result["description"],
        "calories": top_result.get("foodNutrients", [{}])[0].get("value", 0),
        "protein": top_result.get("foodNutrients", [{}])[1].get("value", 0),
        # expand with sodium, fat, etc.
    }

# Full process
meal = "Had boiled eggs and toast with avocado"
foods = extract_food_items(meal)
nutrient_log = [query_usda(food) for food in foods]

Once we log meals as nutrient vectors, we apply a rule-based or learnable scoring engine. For instance:

  • ⚠️ >2400mg sodium/day → flag “high sodium”
  • ✅ >25g fiber/day → “good fiber intake”
  • 🧠 Protein distribution across meals → track muscle-gain optimization
This feedback helps personalize future meals, e.g.:
“Today’s meals were high in sodium. Would you like low-sodium recipes for dinner?”

Engineering Design Notes

  • Latency-aware lookups: Nutrient mapping runs on AWS Lambda for scalability
  • Stateless ingestion: All parsing can be made ephemeral for serverless execution
  • Version control: Food mapping dictionary is versioned to ensure consistency across sessions
